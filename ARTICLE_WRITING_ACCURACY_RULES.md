# 技术文章准确性写作规则

*基于ASM文章修正经验制定 - 2025年7月30日*

## 🚨 核心原则：真实性优先

所有技术文章必须遵循"宁可保守，不可虚构"的原则。每一个数据、每一个声明都必须可验证。

## 📋 强制执行的写作规则

### 1. 数据引用规范

#### ❌ 绝对禁止
- 虚构具体统计数据（如"根据Gartner研究，67%的企业..."）
- 引用不存在的报告（如未来日期的分析报告）
- 使用无法验证的精确百分比（如"提升85%"、"减少73%"）
- 编造具体的时间框架（如"3个月内完成"）
- 虚构公司案例的具体数据

#### ✅ 正确做法
- 使用定性描述："显著提升"、"大幅改善"、"明显减少"
- 引用真实存在的报告，提供可查证的链接
- 使用范围表述："许多企业"、"大量案例"、"部分用户"
- 时间使用阶段性描述："初期阶段"、"中期优化"、"成熟运营"
- 案例使用通用描述："根据供应商案例研究"

### 2. 文章开头免责声明（必须添加）

中文版本：
```markdown
> **注**：本文基于公开信息和行业趋势分析编写，旨在探讨[主题]。具体产品功能和数据请以官方最新信息为准。
```

英文版本：
```markdown
> **Note**: This article is an analytical piece based on publicly available information and industry trends, aimed at exploring [topic]. Specific product features and data should be verified with official sources for the most current information.
```

### 3. 产品功能描述

#### ❌ 避免
- "准确率达到95%以上"
- "处理速度提升10倍"
- "覆盖100%的威胁场景"

#### ✅ 推荐
- "显著提高检测准确率"
- "大幅提升处理效率"
- "覆盖主要威胁场景"

### 4. 案例研究呈现

#### 标准格式
```markdown
**行业案例：**
某[行业]企业面临[挑战描述]的问题。通过实施[解决方案]，
根据供应商提供的案例研究，该企业实现了：
- [定性成果1]
- [定性成果2]
- [定性成果3]
```

### 5. 技术趋势描述

#### 原则
- 基于可观察的市场动向
- 引用真实的技术发展
- 避免过度预测未来

#### 示例
- ❌ "2026年，80%的企业将采用AI驱动的安全方案"
- ✅ "AI技术在安全领域的应用正在快速增长"

### 6. 参考文献标准

#### 必须包含
1. 官方技术博客和公告
2. 公开的技术文档
3. 可验证的新闻报道
4. 学术论文（如有）

#### 格式示例
```markdown
## 参考文献

1. [公司名]. "[文章标题]". 官方博客. [URL]
2. [组织名]. "[文档名称]". 技术文档. [URL]
3. [媒体名]. "[新闻标题]". [日期]. [URL]
```

## 🔍 文章发布前检查清单

### 数据验证（必须完成）
- [ ] 所有统计数据都有可查证的来源
- [ ] 所有引用的报告确实存在
- [ ] 所有百分比数据可以被独立验证
- [ ] 所有时间框架基于真实案例

### 内容审核（必须完成）
- [ ] 已添加文章性质说明/免责声明
- [ ] 产品功能描述使用定性而非定量表述
- [ ] 案例研究标注"基于供应商提供"
- [ ] 技术趋势避免具体预测

### 引用检查（必须完成）
- [ ] 所有引用链接可访问
- [ ] 引用内容与原文一致
- [ ] 参考文献格式规范
- [ ] 删除所有无法验证的引用

## 📊 常用表述对照表

| 避免使用 | 推荐使用 |
|---------|---------|
| 提升85% | 显著提升 |
| 减少90%的错误 | 大幅减少错误率 |
| 100%准确 | 高度准确 |
| 3个月内 | 短期内 |
| 超过10,000家企业 | 众多企业 |
| 据统计，67%... | 许多案例表明... |
| 将在2026年达到... | 预计将持续增长... |

## 🚀 写作流程优化

### 1. 写作前准备
- 收集所有可验证的资料
- 确认数据来源的可靠性
- 准备官方链接和引用

### 2. 写作中注意
- 使用定性描述为主
- 标注所有数据来源
- 保持客观中立态度

### 3. 写作后验证
- 使用检查清单逐项确认
- 请他人帮助事实核查
- 考虑使用AI工具验证可信度

## 💡 经验教训总结

1. **宁缺毋滥**：没有可靠来源的数据，宁可不用
2. **透明诚信**：明确告知读者文章的性质和局限
3. **持续改进**：根据反馈不断优化内容
4. **专业严谨**：保持技术写作的专业标准
