# 技术文章准确性验证检查清单

*最后更新：2025年10月6日*

> **📖 配合使用**：本清单为快速验证工具，详细规则、示例和说明请参考：
> - [《技术文章准确性写作规则》](./ARTICLE_WRITING_ACCURACY_RULES.md)
> - [《Innora技术文档写作风格指南》](./WRITING_STYLE_GUIDE.md)

## 使用说明
本检查清单用于在发布技术文章前进行准确性验证。请逐项勾选，确保所有项目都已完成。

## 一、数据和统计验证 ⚡

### 统计数据检查
- [ ] 所有百分比数据都有明确的来源引用
- [ ] 所有数字统计都可以被独立验证
- [ ] 没有使用"根据XX研究"而无法提供具体链接
- [ ] 没有引用未来日期的报告或研究
- [ ] 所有时间相关数据（X天、X个月）基于真实案例

### 表述方式检查
- [ ] 已将所有"提升X%"改为定性描述
- [ ] 已将所有"减少X%"改为定性描述
- [ ] 已将所有"准确率X%"改为定性描述
- [ ] 已将具体数字（如"15,000个"）改为范围描述
- [ ] 已将具体时间框架改为阶段性描述

## 二、引用和参考文献 📚

### 引用验证
- [ ] 所有引用的报告确实存在（通过搜索验证）
- [ ] 所有引用的链接都可以访问
- [ ] 引用内容与原文保持一致
- [ ] 没有引用不存在的组织或机构
- [ ] 参考文献格式规范统一

### 来源标注
- [ ] 供应商案例都标注了"根据供应商提供"
- [ ] 第三方数据都标注了具体来源
- [ ] 内部观察都标注了"基于行业观察"
- [ ] 预测性内容都使用了适当的限定词

## 三、内容准确性 ✅

### 产品功能描述
- [ ] 产品功能描述与官网信息一致
- [ ] 技术特性没有夸大或虚构
- [ ] 没有声称"业界唯一"或"全球首创"（除非可验证）
- [ ] 使用了恰当的技术术语
- [ ] 避免了绝对化表述

### 案例研究
- [ ] 案例使用了通用行业描述（如"某金融机构"）
- [ ] 没有虚构具体的实施效果数据
- [ ] 成果描述使用定性而非定量方式
- [ ] 标注了案例来源（供应商/公开报道）

## 四、格式和结构 📋

### 必要元素检查
- [ ] 文章开头添加了免责声明（模板见ARTICLE_WRITING_ACCURACY_RULES.md第2节）
- [ ] 包含了执行摘要
- [ ] 有明确的关键词标注
- [ ] 中英文版本内容完全对应
- [ ] 格式符合写作规范

## 五、技术内容验证 🔧

### 技术准确性
- [ ] 技术原理描述准确无误
- [ ] 代码示例经过测试可运行
- [ ] 架构图与文字描述一致
- [ ] 专业术语使用恰当
- [ ] 没有技术性错误

### AI相关内容（如适用）
- [ ] AI技术应用描述基于公开信息
- [ ] 没有夸大AI能力
- [ ] 提到了AI的局限性
- [ ] 使用了恰当的AI术语

## 六、最终审核 🎯

### 整体质量检查
- [ ] 通读全文，确保逻辑通顺
- [ ] 检查是否有遗漏的虚构数据
- [ ] 确认所有修改都已完成
- [ ] 中英文版本修改保持同步
- [ ] 文章价值不因修改而降低

### 发布前确认
- [ ] 已完成所有检查项
- [ ] 已保存最终版本
- [ ] 已准备好应对可能的质疑
- [ ] 已记录所有数据来源备查

## 七、验证工具推荐 🛠️

### 事实核查
- Google Scholar - 验证学术引用
- 官方网站 - 验证产品信息
- Archive.org - 查证历史信息
- 新闻搜索 - 验证事件真实性

### 文本检查
- 搜索引擎 - 验证引用的报告是否存在
- Ctrl+F - 搜索文中的百分比和数字
- 对比工具 - 确保中英文版本一致
