# 技术文章撰写最佳实践指南

## 📚 概述

本指南基于大量技术文章分析和优化经验，旨在帮助创建高质量、专业的安全技术文档。

## 🎯 核心原则

### 1. 真实性优先
- **永远不虚构数据**：宁可使用定性描述，也不虚构具体数字
- **引用可验证来源**：所有数据和案例必须有明确出处
- **标注不确定信息**：对推测性内容明确标注

### 2. 技术深度
- **提供可执行代码**：所有代码示例必须完整且可运行
- **包含技术细节**：深入分析而非表面描述
- **使用真实案例**：基于实际事件和公开报告

### 3. 专业规范
- **术语准确统一**：使用行业标准术语
- **结构清晰完整**：遵循模板结构要求
- **双语质量一致**：中英文版本保持同等专业水准

## ⚠️ 常见错误及纠正

### 1. AI痕迹过重

❌ **错误示例**：
```

```

✅ **正确示例**：
```
该威胁的核心特征包括：
1. 利用供应链攻击实现初始访问
2. 使用合法工具进行横向移动
3. 通过加密通道外传数据
```

✅ **正确示例**：

### 2. 技术细节展示
```python
# 使用代码注释解释关键逻辑
def detect_malicious_npm_package(package_json):
    """检测恶意npm包的特征"""
    
    suspicious_indicators = []
    
    # 检查1：postinstall脚本中的可疑命令
    if "scripts" in package_json:
        if "postinstall" in package_json["scripts"]:
            script = package_json["scripts"]["postinstall"]
            
            # 检测base64编码的命令
            if "base64" in script or "atob" in script:
                suspicious_indicators.append("BASE64_ENCODING")
            
            # 检测网络请求
            if any(cmd in script for cmd in ["curl", "wget", "fetch"]):
                suspicious_indicators.append("NETWORK_REQUEST")
    
    # 检查2：异常的依赖关系
    if "dependencies" in package_json:
        for dep, version in package_json["dependencies"].items():
            # 检测GitHub URL作为依赖
            if "github.com" in version or "git+" in version:
                suspicious_indicators.append("GITHUB_DEPENDENCY")
    
    return suspicious_indicators
```
